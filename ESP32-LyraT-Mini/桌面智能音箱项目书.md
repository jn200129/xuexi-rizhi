## 1. 项目立项

### 1.1 背景与选题

现在的智能音箱有两类：① 小米/小度类的**带屏音箱**——UI 好、功能全，但云端全绑生态、隐私不可控、离线即残废；② 极客向的 Raspberry Pi + HAT——自由度高但硬件成本 500+ 且笨重、外壳难。

LyraT-Mini（约 200 元）+ 1.9" IPS 屏（约 20 元）**接近前者的产品感、后者的可玩性**，同时能做到"离线唤醒 + 云端可换"——今天用百度 API，明天想换 Deepseek 就换，不受生态锁死。这是我要做这个项目的最大动机。

### 1.2 差异化定位

| 对手 | 优势 | 劣势 | 我们如何做 |
|---|---|---|---|
| 小米/小度带屏音箱 | UI 完整、生态大 | 云端强绑定、不可 DIY | 云端服务全部可换、代码全开源 |
| 树莓派 + 屏 | 极高自由度 | 贵、耗电、启动慢、外壳难 | 单板 MCU 秒启动，小巧带电池 |
| Alexa/Google Home | 服务成熟 | 国内不好用 | 全走国内 API |
| 老式桌面收音机 | 情怀 | 无智能 | 保留电台+加智能语音 |

### 1.3 交付预期（MVP）

一台桌面成品（约 100mm × 80mm × 40mm，前面板一屏一喇叭），用户体验：

- **待机（时钟模式）**：大字号时钟 + 星期日期 + 天气图标 + 温度 + 城市；夜间自动降亮度；
- **语音模式**（说"嗨乐鑫"触发）：屏切换到语音波形动画 → 上传语音到 ASR → 显示识别文字 → 显示大模型回复 → TTS 播报；
- **播放模式**（长按 Play 键切入）：
  - 网络电台（默认，Mode 键换台）
  - 蓝牙 A2DP（手机连上直接推音乐）
  - SD 卡本地文件
  - UI：专辑封面（96×96）+ 歌名 + 进度条 + 底部 16 段实时 FFT 频谱；
- **锂电池续航 ≥ 6 小时**（待机 + 偶尔播放和语音交互）；
- Wi-Fi 配网：初次上电扫描无 Wi-Fi 时进入 SoftAP 模式（手机连 `LyraT-Setup` 输入密码）。

### 4.3 FreeRTOS 任务划分

| 任务 | 核心 | 优先级 | 栈 | 职责 |
|---|---|---|---|---|
| `app_main` | 任意 | 5 | 4 KB | 状态机 + 事件总线 |
| `lvgl_task` | Core 0 | 6 | 8 KB | LVGL 心跳 + 渲染 |
| `wakenet_task` | Core 1 | 8 | 8 KB | 音频送 WakeNet |
| `pipeline_task` | Core 1 | 7 | 4 KB | ADF pipeline 事件泵 |
| `fft_task` | Core 1 | 5 | 4 KB | ESP-DSP FFT，出频谱数据 |
| `network_task` | Core 0 | 4 | 8 KB | Wi-Fi + HTTP 客户端 |
| `button_task` | 任意 | 4 | 2 KB | 6 键去抖 |

**分核心原则**：
- **Core 0**：Wi-Fi 协议栈（系统默认）+ LVGL 渲染 + 网络任务；
- **Core 1**：音频 I2S DMA + WakeNet + FFT——避免 Wi-Fi 突发抢 CPU 导致爆音/掉帧。

## 6. 关键技术点解析

### 6.1 LVGL v9 与 ESP-ADF 共存

问题：LVGL 心跳需要固定周期喂 `lv_timer_handler()`，与 ADF pipeline 的事件泵不能互相阻塞。

方案：用 `esp_lvgl_port` 官方组件——它内部帮你处理好任务同步、DMA 缓冲、异步刷新。核心代码：

```c
esp_lcd_panel_handle_t panel;
esp_lcd_new_panel_st7789(io, &panel_cfg, &panel);
lvgl_port_init(&lvgl_cfg);
lv_display_t *disp = lvgl_port_add_disp(&(lvgl_port_display_cfg_t){
    .io_handle = io, .panel_handle = panel,
    .buffer_size = 170 * 40, .double_buffer = true,
});
// 之后 LVGL 心跳、绘制、事件都由 esp_lvgl_port 内部任务处理
```

### 6.2 ASR / LLM / TTS 串接

**每次唤醒后的调用链**：

```
[本地] WakeNet 命中
   ↓
[本地] 录音 → 16k WAV 或 PCM buffer（最长 6 秒 或 VAD 静音）
   ↓
[云端] POST 到 ASR（百度/讯飞）→ 返回文字
   ↓  UI 上屏显示"你说：xxx"
[云端] POST 文字到 LLM（Deepseek/千帆/OpenAI）→ 返回回复文字
   ↓  UI 上屏显示"AI：xxx"
[云端] POST 回复到 TTS（百度/微软）→ 返回 MP3 URL 或数据流
   ↓
[本地] ADF pipeline 播放 MP3 → 出声
```

**同步 vs 流式**：MVP 用同步（等每一步返回），总延时 3–5 秒；后续可切流式 SSE 输出边说边合成，压到 2 秒内。

**Token 与密钥管理**：所有 API Key 放 SD 卡的 `/config/secrets.json`（不进 Git），启动时读入内存；被封了换新 Key 只改文件不烧固件。

### 6.3 FFT 频谱可视化

用 **ESP-DSP** 库的 `dsps_fft2r_fc32`：

```c
esp_err_t err = dsps_fft2r_init_fc32(NULL, N);  // N=1024
// 每 50 ms 从 I2S buffer 拷 1024 采样
dsps_wind_hann_f32(wind, N);   // 汉宁窗
for (int i=0;i<N;i++) fft_in[i*2] = sample[i]*wind[i];
dsps_fft2r_fc32(fft_in, N);
dsps_bit_rev_fc32(fft_in, N);
dsps_cplx2reC_fc32(fft_in, N);
// 幅值取对数 → 分到 16 段 → 送 LVGL 画柱状
```

CPU 消耗：Core 1 上 1024 点 FFT 约 1.5 ms/次，20 Hz 刷新占 3% CPU，完全够。

### 6.4 时钟绘制的性能陷阱

大字号数字（占屏 60% 面积）如果每秒重绘会闪。方案：

- 用 LVGL 的 **partial refresh** —— 只重绘变化的位数（分钟数字比小时数字变化频繁 60 倍）；
- 秒钟用**单独的圆环 canvas** 每秒重画一次，不动大数字；
- 天气图标只在 15 分钟 API 返回后重绘一次。

### 6.5 双电源域与地环路（Hi-Fi 补丁）

给屏加上后，SPI 时钟成为**新噪声源**。工程处理：

- 屏 VCC 走独立 RC 低通滤波（10Ω + 10 μF）；
- SPI 走线远离 I2S 差分；
- 屏和音频不共 3V3 走线（板子上分两路 pad）——即使做不到理想 Hi-Fi，也把可听底噪降到 −70 dBFS 以下。

### 7.2 集成测试（场景）

- **场景 A：桌面待机 12 小时** —— 时钟/天气正确刷新、无 UI 卡顿；
- **场景 B：语音连续交互 20 次** —— 无死机、每次总延时 ≤ 5s；
- **场景 C：切换 3 种音源** —— 电台 → 蓝牙 → SD → 电台，切换无爆音；
- **场景 D：断网播放** —— 电台断线自动切 SD 卡兜底或蓝牙；
- **场景 E：Wi-Fi 弱信号** —— 距离路由器 15m 时电台不断流。

### 7.3 环境测试

- 屋内 50 lux（暗）→ 屏亮度 30% 舒适；
- 屋内 500 lux（白天）→ 屏亮度 100% 可视；
- 环境噪声 30–50 dB SPL 均能唤醒。

## 8. 风险与备选方案

| 风险 | 概率 | 影响 | 备选方案 |
|---|---|---|---|
| LVGL v9 + ADF v2.7 集成问题 | 中 | 高 | 回退 LVGL v8.3（稳定成熟） |
| 屏 SPI 干扰 I2S 音频 | 中 | 中 | 屏走 HSPI，音频独立 I2S bus；屏 VCC 加 π 滤波 |
| Wi-Fi + 蓝牙共存不稳 | 中 | 中 | 蓝牙和 Wi-Fi 分时开（进蓝牙模式关 Wi-Fi） |
| 云端 API 被限流/收费 | 高 | 高 | 抽象层封装，一次实现 3 家（百度/讯飞/Deepseek），运行时切换 |
| PSRAM 不够跑 LVGL 双缓冲 + 播放器 | 中 | 中 | LVGL 用单缓冲 + 部分刷新；专辑封面缩小到 96×96 |
| 大字号中文字库占 Flash | 中 | 低 | 只打包常用 500 字（`--include=常用字库.txt`） |
| 3D 打印外壳散热 | 低 | 低 | 底部开散热槽，靠电池仓热质量兜底 |

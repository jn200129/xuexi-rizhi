一、为什么自己建模

做完《ESP32-LyraT-Mini 桌面智能音箱》项目书之后，主板、屏、喇叭、电池全都定下来了， 但一堆散件放桌上没法叫"成品"——所以要一个外壳。外壳的作用不只是好看：

  • 保护：主板 IO、屏排线、锂电池最脆弱，一撞就废；
  • 声学：喇叭没有腔体等于没喇叭，外壳兼作前腔；
  • 人机：按键要凸出来能戳到、屏要露出来能看到、SD/USB 要方便插；
  • 成本：淘宝买的通用外壳不合尺寸，量身 3D 打印才是唯一路径。

市面上做 3D 建模主流工具是 Fusion 360 / SolidWorks / Blender，图形界面拖拽建。 但对自制硬件项目而言，图形建模的最大问题是「改起来贵」—— 屏换个尺寸、喇叭换个大小、想给后面板加个通风孔，都得重新拖鼠标；改坏了没法回滚。

本文要展示的方案：用 FreeCAD + Python 脚本做参数化建模， 所有尺寸都是变量。想调？改一行数字重跑一遍，1 分钟出新 STL。任何一次改动都在 Git 里留痕， 出错了 git checkout 就回退。硬件项目应该有的开发流程，跟软件项目一样。

二、设计决策

建模之前先把大方向拍板：

表 1  三种外壳风格对比与选择
方案形态优缺
A · 横版桌面式 ✓100×80×40，屏在左、喇叭圆孔在右稳、易插 SD、正视桌面不够"智能音箱"
B · 竖版立式80×60×90，屏在上、喇叭在下最像 HomePod Mini重心难配
C · 复古收音机100×80×60，圆屏+蜂窝网+假旋钮情怀、有辨识度建模复杂、需装饰件

选 A · 横版桌面式。桌面音箱的用户使用场景是"放桌上、坐着看屏"，横版最稳、屏离用户眼睛距离最短。

剩下的技术决策：

  • 分件方式：主体 U 型 + 顶盖两件（前后左右底五面在主体，顶面单独打）——比"上下对半劈"简单，也比"卡扣一体"好装拆；
  • 固定方式：4 颗 M2×10 沉头螺丝拧顶盖——比卡扣可反复维修；
  • 屏/喇叭固定：3M VHB 泡棉双面胶粘前面板背面——比"卡爪"通用（因为不同尺寸的屏 PCB 边缘不同），后期换屏也方便；
  • 散热：前面板麦克风孔阵列 + 底部脚垫抬起 —— 被动散热够用，ESP32 稳定工作只 1.5 W。

四、为什么选 FreeCAD + Python

表 3  3D 建模工具选型对比
工具图形界面脚本化费用评价
Fusion 360强有（Python API 弱）个人免费/商用付费好用但云端强绑
SolidWorks最强宏（VBA）贵过度
Blender强Python 强免费擅长美术模型不擅长精确尺寸
OpenSCAD无（代码 only）专用 DSL免费纯代码派最爱，但学新语法
FreeCAD ✓有完整 Python API免费开源能拖能写、精确尺寸、开源可长期用

选 FreeCAD 的三个决定性理由：

  1. Python API 完整——所有 GUI 能做的事都能用代码做，能命令行 freecadcmd 无头跑；
  2. STEP 双向——参数化改完导出 STEP，别人可用 Fusion/SolidWorks 打开二次编辑；
  3. 之前用过——RK3568 板子外壳（相关笔记暂未成文）就是用它加固底板的，工具链熟。
关键组件叫 CSG（Constructive Solid Geometry）：把复杂形状看作若干简单几何体（长方体、圆柱、圆锥……）通过布尔运算（Union、Difference、Intersection）组合而成。 FreeCAD 的 Part 模块就是 CSG 引擎，"外壳"本质就是"大盒子 减 内腔 减 一堆孔 并 支撑柱"。

五、代码解析（关键片段）

5.1 参数区：一切改动的入口

脚本前 50 行全是常量。想改任何尺寸都改这里，不用碰后面的建模逻辑：

# ========= 参数（按需修改后重跑即得新 STL）=========
W, D, H         = 100.0, 80.0, 40.0   # 外形宽/深/高
WALL            = 2.4                  # 侧壁厚
FLOOR           = 2.0                  # 底板厚
LID_THICK       = 2.5                  # 顶盖厚
LID_STEP        = 2.0                  # 顶盖嵌入深度

# 屏窗口（前面板 Y=0）
SCR_X, SCR_Z    = 15.0, 4.0
SCR_W, SCR_H    = 30.0, 34.0

# 喇叭孔（前面板）
SPK_CX, SPK_CZ  = 72.0, 20.0
SPK_R           = 18.0                 # Ø 36 mm

# 6 键 (右侧 X=W)
BTN_YS          = [12, 24, 36, 48, 60, 72]
BTN_R           = 1.75                 # Ø 3.5

# PCB 支撑柱
PCB_POSTS       = [(18, 25), (82, 25), (18, 55), (82, 55)]
PCB_H, PCB_OD, PCB_ID = 12.0, 5.0, 1.7

5.2 建外盒 + 挖内腔

outer = Part.makeBox(W, D, H)                                    # 100×80×40 实心大盒
inner = Part.makeBox(W-2*WALL, D-2*WALL, H-FLOOR).translate(     # 95.2×75.2×38 内腔
    Base.Vector(WALL, WALL, FLOOR)
)
body = outer.cut(inner)                                          # 减法 = 挖空
# 挖顶部：留台阶给顶盖嵌入
lid_open = box(WALL, WALL, H-LID_STEP, W-2*WALL, D-2*WALL, LID_STEP+0.1)
body = body.cut(lid_open)

三行代码就把整个 U 型主体的"骨架"做出来了。cut() 就是 CSG 的减法，返回新形状——原来的 outer、inner 不会被修改（immutable 风格）。

5.3 循环打孔（6 键）

# 6 键孔：轴 // X（从右侧 X=W 面挖进去）
for y in BTN_YS:
    hole = cyl_x(y, BTN_Z, W-WALL-0.1, BTN_R, WALL+0.2)
    body = body.cut(hole)

Python 循环的威力：想加两排？改成两个 for；想改间距？改 BTN_YS 数组；想改半径？改 BTN_R——完全不用碰这段建模代码。

5.4 加支撑柱（fuse 加法）

# PCB 支撑柱：外柱 - 内孔 = 空心柱
for cx, cy in PCB_POSTS:
    post = cyl_z(cx, cy, FLOOR, PCB_OD/2, PCB_H)
    hole = cyl_z(cx, cy, FLOOR, PCB_ID/2, PCB_H+0.1)
    post = post.cut(hole)          # 先在柱心挖 M2 自攻孔
    body = body.fuse(post)          # 再合并到主体上（fuse 就是 CSG 加法）

+0.1 是关键小细节——布尔运算时两个面完全重合会产生"零厚度面"，STL 无法正确表达。刻意让被减的部分伸出一点点（0.1 mm 远小于打印精度），保证结果拓扑正确。

5.5 顶盖沉头孔

for cx, cy in LID_POSTS:
    shaft = cyl_z(cx, cy, H-LID_STEP-1, LID_SCREW_SHAFT_D/2, LID_STEP+2)
    lid = lid.cut(shaft)                                        # 螺丝穿孔
    head = cyl_z(cx, cy, H-LID_SCREW_HEAD_H, LID_SCREW_HEAD_D/2,
                 LID_SCREW_HEAD_H+0.1)
    lid = lid.cut(head)                                          # 顶部沉头凹槽

沉头 = 大孔在上、小孔穿透——两次 cut 就够。M2×10 沉头螺丝装完与顶盖平齐，看不见钉头。

5.6 导出

def export(shape, name):
    shape.exportStep(f"{OUT_DIR}/{name}.step")                    # STEP（可编辑）
    mesh = MeshPart.meshFromShape(shape,                          # STL（可打印）
                                  LinearDeflection=0.15,
                                  AngularDeflection=0.4)
    mesh.write(f"{OUT_DIR}/{name}.stl")

LinearDeflection=0.15 控制曲面三角化精度：0.15 mm 弦高误差在 3D 打印 0.2 mm 层高之内，完全不损失细节。

5.7 命令行运行

D:\FreeCAD\FreeCAD_1.1.1-Windows-x86_64-py311\bin\freecadcmd.exe case_generator.py

无头模式（不开 GUI）跑完 1 分钟出 3 个 STL、3 个 STEP。整个过程可以塞进 CI 里跑单元测试。

六、几何校验：valid + closed = 可打印

做完 STL 别急着切片，先跑一遍几何校验：

for name in ["main", "lid", "assembly"]:
    shape = Part.Shape()
    shape.read(f"D:\\lyrat_mini_case\\output\\LyraT_case_{name}.step")
    bb = shape.BoundBox
    print(f"[{name}] {bb.XLength}×{bb.YLength}×{bb.ZLength}")
    print(f"  Volume {shape.Volume/1000:.2f} cm^3")
    print(f"  Closed {shape.isClosed()}   Valid {shape.isValid()}")
表 4  3 个 STEP 的几何校验结果
部件包围盒体积面数ClosedValid
主体100×80×4046.61 cm³67✓✓
顶盖100×80×2.519.89 cm³18✓✓
装配预览100×80×4064.43 cm³83✓✓
Closed + Valid 是打印的必要条件。Closed 意思是"表面完全封闭无漏洞"，Valid 意思是"拓扑合法无自交无重复面"。 不 closed 的模型切片器会当作"敞开的壳"处理，出现意料之外的支撑或空腔。用参数化脚本产出的模型天然通过——因为所有布尔运算 FreeCAD 内部就做了拓扑检查。

八、组装步骤

  1. 主体打印完 → 清除支撑（前面板屏窗口内可能有支撑残留，用美工刀修边）；
  2. 固定 LyraT-Mini 主板到 4 根 PCB 支撑柱（M2×5 mm 自攻螺丝）；
  3. 接屏排线（SPI 6P）、喇叭线、麦克风延长线、电池 XH2.54 头；
  4. 屏模块贴 3M VHB 泡棉双面胶粘在前面板屏窗口背面（对齐显示区）；
  5. 喇叭同样用泡棉胶粘在前面板喇叭孔背面；
  6. 电池贴双面胶固定在底板前段（主板下方留出的空间）；
  7. 检查所有线材没压到 PCB 边缘、没跨过螺柱；
  8. 盖顶盖 → 4 颗 M2×10 mm 沉头螺丝（十字或内六角均可）拧入 4 根螺柱；
  9. 底部贴 4 颗 Ø 8 mm 硅胶脚垫（可选，防滑防划桌面）。
装配顺序有讲究：先接排线再粘屏——不然屏粘死了排线插不上去。锂电池最后接，避免过程中短路。

九、二次修改：改一行数字重生 STL

这就是参数化建模的价值。给你几个常见修改场景：

表 6  常见修改一览
场景改哪个参数
换更小的屏（1.14"）SCR_W = 22, SCR_H = 26
换更大的喇叭（Ø 50）SPK_R = 23；SPK_CX, SPK_CZ 顺便调居中
按键改成 4 个BTN_YS = [16, 32, 48, 64]
整体加高（塞更大电池）H = 55，其余不动
后面板加通风孔复用 MIC_ARR_* 代码，把 Y 从 -0.1 改成 D-WALL-0.1
加挂墙孔在主体后面板加两个 Φ 4 cyl_y
墙加厚（强度更强）WALL = 3.2

改完命令行重跑：

D:\FreeCAD\FreeCAD_1.1.1-Windows-x86_64-py311\bin\freecadcmd.exe case_generator.py

约 1 分钟就出新 STL。整个流程可以塞进 Makefile / package.json scripts：

build:
	freecadcmd case_generator.py
	python verify.py         # 几何校验
	@echo "STL ready: output/*.stl"
硬件项目的软件工程范式：
  1. 参数化脚本 → 版本控制（Git）；
  2. 命令行工具链 → 一键构建（Makefile/CI）；
  3. 自动校验 → 单元测试；
  4. 回滚方便 → git checkout 撤回；
  5. 协作友好 → 别人拉代码就能跑出一样的 STL。
比"Fusion 里保存一个 .f3d 文件发给别人"专业得多。

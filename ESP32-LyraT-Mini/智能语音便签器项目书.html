0. 项目一句话概括

「一台离线可用的语音便签器：喊唤醒词就能录下想法，按键回放，SD 卡本地存储，Wi-Fi 连网时自动同步到自建服务器。」

离线唤醒 VAD 自动断句 AMR 压缩 SD 存储 按键回放 云同步 电池续航 ≥ 8h 隐私优先

1. 项目立项

1.1 背景与选题

灵感/备忘随时会冒出来，掏手机打字/录音门槛都比想象中高——尤其在走路、开车、做饭时。 桌面式语音便签器（一按就录、说完就完、要用时回放）能真正做到「零操作成本」的想法收集， 是被小众硬件市场（Rocketbook Beacon、Bushnell Notebot）反复验证过的形态。

现有智能音箱不适合这个场景：① 依赖云识别 → 断网即残废、隐私不可控； ② 唤醒后必须"回答"某个问题 → 不支持"我只是想记一句话"的自由留言模式。

本项目要做的差异化：所有语音数据默认只存在本地 SD 卡（隐私优先）， 云同步是可选功能；离线 WakeNet 唤醒 + VAD 自动收尾，全程双手无需接触设备。

1.3 交付预期（MVP）

一台带外壳的成品便签器，用户拿到手能做的事：

  1. 上电听到"叮"提示音 → 系统就绪；
  2. 说 "嗨乐鑫"（唤醒词）→ LED 变绿 → 开始录音，说完停顿 1.5 秒自动结束；
  3. 按 Play 键：从最新一条便签开始按顺序播放；
  4. 按 Mode 键：切换到"上一条/下一条"浏览；
  5. 长按 Set 键：删除当前便签；
  6. 有 Wi-Fi 时后台自动上传新便签到自建 HTTP 服务器（POST AMR 文件）；
  7. 锂电池续航 ≥ 8 小时（待唤醒 + 偶尔录放）。

6. 关键技术点解析

6.1 WakeNet + VAD 的协同

WakeNet 是持续检测（每帧滑窗），VAD 是"人声有/无"的粗判。二者用法：

  • 待唤醒态：只跑 WakeNet，检测到唤醒词 → 转录音态；
  • 录音态：关闭 WakeNet（省 CPU）、开 VAD；连续 1.5 秒判无人声 → 结束录音；
  • 超时保护：无论 VAD 结果，60 秒硬截止（防止风扇噪声被判成人声无限录下去）。

ESP-ADF 里 esp_sr 组件的用法：

srmodel_list_t *models = esp_srmodel_init("model");
esp_wn_iface_t *wakenet = (esp_wn_iface_t *)esp_wn_handle_from_name(
    esp_srmodel_filter(models, ESP_WN_PREFIX, NULL));
model_iface_data_t *model = wakenet->create(
    esp_srmodel_filter(models, ESP_WN_PREFIX, NULL), DET_MODE_2CH_90);
// 每帧送数据：
int r = wakenet->detect(model, i2s_buffer);
if (r > 0) { /* 唤醒词命中 */ }

6.2 ADF pipeline 组装（录音路径）

audio_pipeline_handle_t rec_pipe = audio_pipeline_init(&pcfg);
audio_element_handle_t i2s_reader = i2s_stream_init(&i2s_read_cfg);
audio_element_handle_t amr_enc    = amr_encoder_init(&amr_cfg);
audio_element_handle_t fs_writer  = fatfs_stream_init(&fs_write_cfg);
audio_pipeline_register(rec_pipe, i2s_reader, "i2s");
audio_pipeline_register(rec_pipe, amr_enc,    "enc");
audio_pipeline_register(rec_pipe, fs_writer,  "fs");
audio_pipeline_link(rec_pipe, (const char*[]){"i2s","enc","fs"}, 3);
audio_element_set_uri(fs_writer, note_path);
audio_pipeline_run(rec_pipe);

播放路径把 3 个元素反过来（fatfs_stream → amr_decoder → i2s_stream）+ 换 I2S 方向即可。

6.3 SD 卡「掉电不丢数据」

  1. 每写完一帧 AMR（20 ms 音频 = 32 字节）后不立刻 sync，但每 5 秒 f_sync() 一次；
  2. 元数据 index.json 采用双缓冲写：先写 index.json.tmp → sync → rename；
  3. 加一个 GPIO 检测充电断开 → 立刻 flush + fclose；
  4. 启动时扫描 /notes/ 里没在 index.json 里的孤儿 .amr → 自动恢复。

6.4 电池续航优化

表 6-1  各场景平均电流预算（3.7V 单节锂电池）
场景平均电流优化手段
待唤醒60 mAWi-Fi 关；CPU DFS 降至 80 MHz；ES7243 常开；WakeNet 常跑
录音中90 mA同上 + SD 写
播放中130 mA+ 功放 + Codec 播放
Wi-Fi 上传中240 mA（峰值 500 mA）集中批量上传（一小时一次）

esp_pm_configure() 打开 DFS + Light Sleep（保持外设唤醒源）。1000 mAh 电池：60 mA × 8h = 480 mAh，剩 500 mAh 余量给录放和上传峰值——理论续航 ≥ 10 小时。

7.2 集成测试（场景）

  • 场景 A：办公室连续 8 小时使用——记 20 条便签，全部可播、可上传、电池剩余 ≥ 20%；
  • 场景 B：走路时手持——WakeNet 命中率不低于办公室场景 -20%；
  • 场景 C：拔电测试——录音中拔电 3 次、播放中拔电 3 次，重启后所有文件仍能播放；
  • 场景 D：SD 卡快满——SD 剩余 < 100 MB 时提示、拒绝新录音、允许播放已有的。

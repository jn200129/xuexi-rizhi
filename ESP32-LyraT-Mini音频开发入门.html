一、认识这块板子：ESP32-LyraT-Mini V1.2

ESP32-LyraT-Mini 是乐鑫官方的单声道音频开发板，定位是「用最小的成本把 ESP32 音频应用链路跑通」： 一颗 ESP32 模组 + 一收一放两颗音频芯片 + 麦克风/喇叭/耳机/SD 卡/按键/锂电池管理，智能音箱该有的硬件它都有。 官方音频框架 ESP-ADF（Espressif Audio Development Framework）对它开箱即用。

1.1 硬件组成

表 1  ESP32-LyraT-Mini V1.2 主要硬件
部件型号/规格作用
主控模组ESP32-WROVER-E（双核 240 MHz，4 MB Flash + 8 MB PSRAM，Wi-Fi + 蓝牙双模）跑协议栈、解码器、语音模型——8 MB PSRAM 是能跑语音识别/网络流媒体的关键
播放通路ES8311 低功耗单声道编解码器（I2S 数据 + I2C 配置）DAC 放音，接 3.5 mm 耳机座（带插入检测）和功放 → 2.0 mm 喇叭座（推荐 4 Ω 3 W）
录音通路ES7243 音频 ADC + 板载单麦克风拾音，走独立 I2S 送进 ESP32——录音、语音识别都靠它
存储MicroSD 卡槽（1 线模式）存放/播放本地音乐、保存录音
人机交互6 个功能键（Rec / Mode / Play / Set / Vol− / Vol+）+ Boot / RST播放器控制、录音触发；Boot+RST 进下载模式
USB / 调试两个 Micro-USB：UART 口（CP2102N 桥）+ 供电口烧录、串口日志、供电
电源管理AP5056 锂电池线性充电芯片5 V USB 或 3.7 V 锂电池（建议 >1000 mAh）供电，可边充边用；音频与数字电源分开稳压、隔离噪声
和它的兄弟们：LyraT（双声道 ES8388、双麦）→ LyraT-Mini（单声道、单麦、更便宜）→ Korvo 系列（麦克风阵列，远场语音）。 Mini 的定位是入门与量产原型：近场语音 + 播放的项目它全能做；要「三米外喊小度」那种远场唤醒，才需要上麦克风阵列的板子。

二、环境搭建（Windows 实操）

官方文档的开发环境 = ESP-IDF（ESP32 基础开发框架，编译器/烧录器都在这）+ ESP-ADF（音频框架，以 IDF 为地基）。 好消息是 ESP-IDF 不用单独装——v2.7 的 esp-adf 把配套版本的 esp-idf 作为子模块内置了，一个仓库全搞定。以下是本机的实际操作记录。

2.1 版本选择：v2.7 还是 master？

表 2  ESP-ADF 两条版本线（2026-07 现状）
版本架构配套 IDF选择建议
v2.7（正式版）经典 audio_pipeline/element 架构，examples/ 下 60+ 例程，板级支持齐全内置 esp-idf 子模块（v5.4 系）入门选它——官方文档、教程、例程全部对应这条线
master（v3.0 开发版）全新 ESP-GMF 通用多媒体框架，目录变为 adf_examples/需自装 ESP-IDF ≥ v5.5.2新架构方向，但资料还少，暂不建议入门用

2.2 实际执行的命令

前置：Windows 已有 git 和 Python（3.9~3.13 皆可；install 脚本会另建独立的 Python 虚拟环境，不污染系统）。全部文件按惯例放 D 盘：

# ① 克隆 esp-adf 官方仓库（先不加 --recursive，切好版本再拉子模块，省流量）
mkdir D:\esp && cd D:\esp
git clone https://github.com/espressif/esp-adf.git
cd esp-adf

# ② 切到 v2.7 正式版
git checkout v2.7

# ③ 递归拉取子模块——最大的一步，esp-idf 全量约 2 GB
git submodule update --init --recursive
#   网络不稳时：加 --depth 100 浅克隆大幅减量；失败了重跑同一条命令即可，
#   已完成的子模块会自动跳过（相当于断点续传）

# ④ 安装工具链（编译器、烧录工具、独立 Python 虚拟环境）
install.bat
#   下载慢可先设乐鑫官方 CDN 再跑：set IDF_GITHUB_ASSETS=dl.espressif.cn/github_assets

# ⑤ 进入开发环境（设好 IDF_PATH、ADF_PATH、PATH——每开一个新终端都要执行）
export.bat
Linux/macOS 对应 ./install.sh 与 . ./export.sh，其余完全一致。 嫌每次 export 麻烦，可以给「已 export 的 cmd」做个快捷方式（参考官方 ESP-IDF CMD 的做法： cmd /k "cd /d D:\esp\esp-adf && export.bat"）。

三、跑通第一个例子：play_mp3_control

官方指定的 Hello World 是 examples/get-started/play_mp3_control——播放一段内置 MP3，按键控制播放/暂停/音量。流程是所有 ADF 项目通用的：

cd examples\get-started\play_mp3_control

idf.py set-target esp32          # 选芯片（LyraT-Mini 是经典 ESP32）
idf.py menuconfig                # 关键一步：选板子！
#   Audio HAL → Audio board → ESP32-Lyrat-Mini V1.1
#  （V1.2 的板子同样选这项，官方就是这么对应的）

idf.py build                     # 编译
idf.py -p COM口 flash monitor    # 烧录 + 打开串口监视器（板子接「UART」口）
#   退出 monitor：Ctrl + ]

接上耳机或喇叭，复位后就能听到音乐；串口里能看到 pipeline 各元素的状态日志。 验证录音通路可以再烧 examples/recorder/pipeline_wav_amr_sdcard（按 Rec 键录到 SD 卡）。

menuconfig 选板子为什么重要：ADF 把每块官方板的引脚、Codec 型号、功放使能脚都封装成了「board」组件（components/audio_board/lyrat_mini_v1_1/）。 选对板子，audio_board_init() 一行就把 ES8311+ES7243 全配好；选错则无声/无麦。自己画的板子也可以照这个目录结构加一块「自定义 board」。

四、ESP-ADF 核心概念：音频管道

ADF 的编程模型一句话：把音频处理拆成一个个「元素」（element），用「管道」（pipeline）串起来，数据在环形缓冲里自动流动。 你不用手写「读文件→解码→重采样→喂 I2S」的循环，只需声明链路、监听事件。

在这套管道之上，ADF 还提供成品级「服务」组件：Wi-Fi 配网服务、蓝牙服务、按键外设、ESP Audio 播放器（多格式一键播放）、语音识别接口（对接 esp-sr 的 WakeNet 唤醒词 / MultiNet 命令词模型）等——例程基本就是这些积木的不同拼法。

由易到难的一条学习路线

结合板子特性的项目点子

  • 语音控制开关：wwe 唤醒 + MultiNet 离线命令词（「打开灯」「关闭风扇」）+ GPIO 继电器——全离线、断网可用，最能出成果的实战。
  • 宝宝故事机 / 白噪声机：SD 卡 + 6 个实体按键选曲 + 锂电池——板子的按键和电源管理天生为此准备。
  • 网络收音机：living_stream 例程 + 电台 HLS 直播源列表，Mode 键换台。
  • SIP 对讲门铃：voip 例程，Rec 键呼叫，手机 SIP 客户端应答——一收一放两路 I2S + AEC 正好用上。
  • 会议录音笔：VAD 自动启停 + AMR 压缩存 SD + 充电电池，录一整天。
  • 老人呼叫器：按键触发录音上云 + 微信/Telegram Bot 推送（cloud_services 思路）。
  • 音乐律动灯：led_pixels 例程接 WS2812 灯带，播放器变氛围灯。
  • 接大模型的 AI 音箱：唤醒 → 录音 POST 到自建服务（语音转文字 → LLM → 文字转语音）→ 拉回 MP3 播放——DuerOS 链路的「自主可控版」。
单麦的边界要心里有数：WakeNet 在 LyraT-Mini 上做近场（1~2 米）唤醒没问题；嘈杂环境远场唤醒、声源定位需要麦克风阵列（Korvo 系列）。 另外 ESP32 经典款不支持 MultiNet 最新的大模型命令词（那些要 ESP32-S3），入门离线命令词够用即可。

六、踩坑与注意事项

1. master ≠ 文档里的 ADF
2026 年起 master 已切到 v3.0 新架构（ESP-GMF，目录是 adf_examples/，且要求 IDF ≥ 5.5.2）。照官方入门文档学习务必 git checkout v2.7，否则连 examples/ 目录都找不到。
2. 子模块下载中断
esp-idf 子模块体量约 2 GB，连接不稳时容易中途失败。对策：重跑 git submodule update --init --recursive（已完成的模块自动跳过），或加 --depth 100 浅克隆减小传输量；也可以设 git config http.lowSpeedLimit 1000、http.lowSpeedTime 60 让慢连接尽早失败重试。
3. 工具链下载慢
install 前设 IDF_GITHUB_ASSETS=dl.espressif.cn/github_assets（乐鑫自家 CDN，非第三方镜像），pip 下载慢可换国内源。
4. 串口驱动与端口
板上是 CP2102N，Win10/11 一般免驱；设备管理器看 COM 号。烧录/日志都接标着 UART 的那个 Micro-USB 口（另一个只供电）。烧录失败就按住 Boot 再点 RST 进下载模式。
5. menuconfig 忘选板子
现象是编译能过、烧完没声/麦不响。Audio HAL → Audio board → ESP32-Lyrat-Mini V1.1（V1.2 硬件也选它）。
6. export 不是一劳永逸
export.bat 只对当前终端生效，每开新窗口都要跑一次；两个框架路径由它设置（IDF_PATH=D:\esp\esp-adf\esp-idf、ADF_PATH=D:\esp\esp-adf）。
7. 杀毒软件干扰
官方明确提示安装工具链时可能被杀软拦截（大量小文件+改 PATH），install 报奇怪的权限错误时先临时关掉。
